Techniques de prompting

5.1 Les 4 principes fondamentaux

Principe 1 : Commencer large, puis préciser

❌ "Fais une validation"

✅ "Crée une méthode C# qui valide une adresse email.

- Entrée : string (peut être null)
- Sortie : bool
- Utiliser une regex ou MailAddress
- Retourner false si null ou vide

Principe 2 : Fournir des exemples

"Crée une fonction qui extrait les mentions d'un tweet.
 Entrée : '@jean et @marie vont au cinéma'
 Sortie attendue : ['@jean', '@marie']"

Principe 3 : Décomposer les tâches complexes

Au lieu de "Crée une API d'authentification" :

  1. "Crée l'entité User avec Email et PasswordHash"
  2. "Ajoute le endpoint POST /auth/register"
  3. "Implémente POST /auth/login avec génération JWT"
  4. "Crée le middleware de validation du token"

Principe 4 : Utiliser le contexte

  • Ouvrir les fichiers liés avant de prompter
  • Sélectionner le code concerné
  • Utiliser #file:UserService.cs ou @workspace

5.3 Patterns de prompts réutilisables

Refactoring

Refactorise ce code pour :
- Extraire la logique dupliquée
- Améliorer les noms de variables
- Réduire la complexité
- Garder le même comportement

Debug

Erreur : [coller l'erreur]
Code : [sélection ou #file]
Comportement attendu : [description]

Identifie la cause et propose une correction.

Génération de tests

Génère des tests unitaires pour cette classe.
- Couvrir les cas nominaux
- Couvrir les edge cases (null, vide, limites)
- Mocker les dépendances
- Utiliser le pattern AAA

Revue de code

Fais une revue de ce code en vérifiant :
1. Bugs potentiels et edge cases
2. Failles de sécurité
3. Problèmes de performance
4. Respect des conventions du projet

Pour chaque issue : localisation, gravité, suggestion de fix.

Migration / Mise à jour

Migre ce code de [ancien] vers [nouveau].
- Préserver le comportement existant
- Utiliser les nouvelles APIs recommandées
- Signaler les breaking changes
